# 시나리오 01: 스타트업의 첫 번째 위기

> 창업 2년 차 스타트업이 존폐의 기로에 섰습니다.
> 투자자, 창업자, 직원, 고객의 입장에서
> 최선의 판단을 내려야 합니다.
> 정답은 없습니다. 판단의 근거가 중요합니다.

---

## 배경

### 회사 소개

**회사명**: 케어링크 (CareLink)
**업종**: 헬스케어 스타트업
**설립**: 2년 전
**서비스**: 노인 돌봄 매칭 플랫폼
- 돌봄 제공자(요양보호사)와 가족을 연결
- 앱 기반 실시간 돌봄 상황 공유
- 현재 서울·경기 지역 서비스 중

**현재 현황**:
```
누적 사용자:  12,000명
월간 활성 사용자: 3,200명
월 매출:  4,800만 원
월 비용:  8,200만 원
월 적자:  3,400만 원
보유 현금:  1억 2,000만 원
런웨이:  약 3.5개월
```

## 사건의 발단

3개월 전 Series A 투자 유치를 추진했으나
최종 단계에서 주요 투자사가 투자를 철회했습니다.

**투자 철회 이유:**
- 시장 성장 속도가 예상보다 느림
- 수익화 모델의 불확실성
- 경쟁사(대형 플랫폼)의 유사 서비스 출시 예고

현재 3.5개월 치 현금만 남은 상태에서
창업팀은 긴급 경영진 회의를 소집했습니다.

## 현재 상황

### 내부 상황

**창업팀 구성:**
```
대표이사 김민준 (34세):
  배경: 사회복지학 전공, 요양원 근무 경험
  성향: 사회적 가치 중시, 수익보다 임팩트 우선
  현황: 지분 45%, 무보수 근무 중

CTO 이서연 (31세):
  배경: 컴퓨터공학, 대기업 개발자 출신
  성향: 기술 완성도 중시, 현실적 판단
  현황: 지분 25%, 시장 평균 대비 60% 급여

COO 박지호 (36세):
  배경: 컨설팅 출신, 운영 전문
  성향: 효율성 중시, 숫자 기반 판단
  현황: 지분 20%, 시장 평균 대비 60% 급여
```

**직원 현황:**
```
전체 직원: 18명
개발팀: 7명
운영팀: 6명
영업/마케팅: 5명
평균 급여: 월 320만 원 (시장 대비 70%)
```

**직원들의 현재 심리:**
- 급여가 낮음에도 비전을 믿고 함께함
- 최근 투자 철회 소식 후 이직 문의 증가
- 핵심 개발자 2명이 이직 제안 받은 상태

### 외부 상황

**경쟁 환경:**
```
신규 경쟁자 등장:
  네이버케어: 대형 플랫폼의 유사 서비스
  → 6개월 후 출시 예정
  → 막대한 마케팅 예산 보유

기존 경쟁자:
  케어네이버: 업계 1위, 시장 점유율 35%
  돌봄넷: 업계 2위, 시장 점유율 22%
  케어링크: 현재 점유율 8%
```

**잠재 투자자 상황:**
```
투자자 A (임팩트 투자사):
  관심: 사회적 가치, 노인 돌봄 임팩트
  조건: 수익성보다 사회적 지표 중시
  투자 가능 금액: 5억 원
  조건: 지분 20% + 이사회 참여

투자자 B (일반 VC):
  관심: 수익성, 성장성
  조건: 6개월 내 흑자 전환 계획 필요
  투자 가능 금액: 10억 원
  조건: 지분 30% + 경영 개입권

투자자 C (전략적 투자자, 요양병원 그룹):
  관심: 자사 사업과의 시너지
  조건: 자사 환자 데이터 공유 요구
  투자 가능 금액: 8억 원
  조건: 지분 25% + 데이터 독점 계약
```

## 핵심 딜레마

경영진 회의에서 세 가지 방향이 논의되고 있습니다.

### 옵션 A: 긴축 운영 + 임팩트 투자 유치

```
내용:
  직원 30% 감원 (18명 → 12-13명)
  임팩트 투자자 A로부터 5억 원 유치
  사회적 가치 중심 운영 유지

예상 결과:
  런웨이: 약 12개월 확보
  월 적자: 3,400만 → 1,500만 원으로 감소
  팀 사기 타격 불가피
  사회적 미션 유지 가능

주요 리스크:
  핵심 인력 이탈 가능성
  서비스 품질 저하
  경쟁사 출시 전 대응 역량 약화
```

### 옵션 B: 피벗 + VC 투자 유치

```
내용:
  B2C → B2B로 피벗
  (개인 가족 대상 → 요양원·병원 대상)
  VC 투자자 B로부터 10억 원 유치
  6개월 내 흑자 전환 목표

예상 결과:
  런웨이: 약 18개월 확보
  새로운 수익 모델 구축 가능
  기존 사용자 기반 일부 손실

주요 리스크:
  피벗 성공 여부 불확실
  VC의 경영 개입으로 방향성 충돌 가능
  기존 미션(노인 가족 지원)에서 이탈
```

### 옵션 C: 전략적 투자 + 데이터 협력

```
내용:
  요양병원 그룹과 전략적 파트너십
  8억 원 투자 유치
  환자 데이터 공유 및 서비스 통합

예상 결과:
  런웨이: 약 15개월 확보
  즉각적인 B2B 매출 발생 가능
  대형 네트워크 활용 가능

주요 리스크:
  개인정보·데이터 윤리 문제
  투자자 종속 위험
  서비스 독립성 상실
```

## 역할 배정

### 3인 진행 시

```
역할 1: 대표이사 김민준
  목표: 사회적 미션 유지 + 회사 생존
  고민: 직원들에 대한 책임감
  성향: 임팩트 우선

역할 2: CTO 이서연
  목표: 기술 팀 보호 + 현실적 생존
  고민: 핵심 개발자 이탈 방지
  성향: 현실적 판단 우선

역할 3: 투자자 B (VC)
  목표: 투자금 회수 + 수익 극대화
  고민: 리스크 최소화
  성향: 수익성 우선
```

### 4인 진행 시

```
위 3개 역할 + 다음 추가:

역할 4: 핵심 개발자 최유나
  목표: 커리어 보호 + 의미 있는 일
  고민: 이직 제안 수락 여부
  성향: 성장 가능성 + 워라밸 중시
```

### 5인 진행 시

```
위 4개 역할 + 다음 추가:

역할 5: COO 박지호
  목표: 운영 효율화 + 생존 가능성 극대화
  고민: 감원 집행의 현실적 어려움
  성향: 숫자 기반 냉정한 판단
```

### 6인 진행 시

```
위 5개 역할 + 다음 추가:

역할 6: 고객 대표 (장기 사용자 가족)
  배경: 치매 부모님을 케어링크로 관리 중
  목표: 서비스 유지 + 품질 보장
  고민: 서비스 중단 시 대안 없음
  성향: 사용자 입장의 현실적 요구
```

## 진행 방법

### 1단계: 배경 파악 (15분)

모든 참여자가 시나리오를 읽고
각자의 역할과 이해관계를 파악합니다.

```
확인 사항:
  □ 회사의 현재 재정 상황
  □ 각 옵션의 장단점
  □ 자신의 역할과 목표
  □ 다른 역할과의 이해관계 충돌
```

### 2단계: 입장 정리 (10분)

각 참여자가 자신의 역할 관점에서
선호하는 옵션과 그 이유를 개인적으로 정리합니다.

```
정리 항목:
  선호 옵션: A / B / C / 기타
  핵심 이유: (2-3가지)
  절대 수용 불가 조건:
  협상 가능한 부분:
```

### 3단계: 자유 협상 (30-40분)

역할에 몰입해서 자유롭게 협상하고 토론합니다.

```
진행 방식:
  - 각자 입장 발표 (역할당 2-3분)
  - 자유 토론 및 협상
  - 조건 제시 및 수정
  - 최종 합의 시도

진행자 역할:
  - 시간 관리
  - 교착 상태 시 새로운 정보 투입
  - 발언 기회 균등 조율
```

**진행자용 추가 이벤트 카드:**

```
이벤트 1 (협상 20분 후 투입 가능):
  "경쟁사 네이버케어가 출시를
   3개월 앞당긴다는 뉴스가 나왔습니다."
  → 시간 압박 증가

이벤트 2 (교착 상태 시 투입):
  "핵심 개발자 최유나가
   경쟁사로부터 연봉 2배 제안을 받았습니다.
   오늘 결정을 원합니다."
  → 긴박감 증가

이벤트 3 (선택적 투입):
  "정부에서 노인 돌봄 플랫폼에
   사회적기업 지원금 3억 원 공모를 발표했습니다."
  → 새로운 옵션 등장
```

### 4단계: 결정 선언 (5분)

협상 결과를 정리하고 최종 결정을 선언합니다.

```
합의 도달 시:
  - 선택한 옵션과 수정 조건 발표
  - 각 역할의 수용 여부 확인

합의 실패 시:
  - 합의 실패 자체도 유효한 결과
  - 왜 합의가 안 됐는지 분석
```

### 5단계: 성찰 토론 (15-20분)

역할에서 벗어나 시뮬레이션을 돌아봅니다.

```
토론 질문:
  ①어떤 가치가 충돌했나요?
    (수익 vs 미션, 성장 vs 안정, 개인 vs 조직)

  ②실제 이런 상황이라면 어떻게 했을까요?

  ③어떤 역할이 가장 어려웠나요? 왜?

  ④현실에서 스타트업이 이런 상황을 어떻게
    극복했는지 아는 사례가 있나요?

  ⑤이 시뮬레이션에서 배운 것은 무엇인가요?
```

## 진행자 가이드

### 효과적인 진행을 위한 팁

**역할 몰입 유도:**
```
"지금은 [역할명]의 입장에서 말씀해 주세요."
"[역할명]이라면 이 상황에서 어떤 감정이 들까요?"
"[역할명]의 가장 큰 두려움은 무엇인가요?"
```

**교착 상태 해소:**
```
"지금 가장 중요한 한 가지만 말씀해 주신다면?"
"절대 양보할 수 없는 것은 무엇인가요?"
"상대방이 무엇을 원하는지 이해하셨나요?"
```

**깊이 있는 토론 유도:**
```
"왜 그렇게 생각하시나요?"
"다른 역할의 입장도 들어볼까요?"
"현실에서는 이런 결정이 어떤 결과를 낳았나요?"
```

### 주의사항

```
①정답 제시 금지:
  진행자는 특정 옵션이 더 낫다고 유도하지 않습니다.

②모든 판단 존중:
  어떤 결정도 그 자체로 가치 있습니다.

③역할 이탈 주의:
  참여자가 자신의 의견을 말하면
  역할로 돌아오도록 유도합니다.

④시간 관리:
  성찰 토론 시간은 반드시 확보합니다.
  이것이 시뮬레이션의 핵심입니다.
```

## 실제 사례 참고

### 유사 사례 1: 당근마켓의 피벗

```
상황: 초기 하이퍼로컬 커뮤니티 앱
위기: 수익 모델 불확실, 투자 압박
결정: B2C 광고 모델 강화 + 지역 밀착 유지
결과: 사용자 기반 유지하며 수익화 성공
교훈: 핵심 가치(로컬 커뮤니티) 유지가 핵심
```

### 유사 사례 2: 쏘카의 구조조정

```
상황: 코로나19로 매출 급감
위기: 런웨이 부족, 투자 유치 어려움
결정: 직원 20% 감원 + 핵심 사업 집중
결과: 생존 후 IPO 성공
교훈: 고통스러운 결정이 장기 생존의 기반
```

### 유사 사례 3: 토스의 방향 전환

```
상황: 초기 인증서 간편 서비스
위기: 금융당국 규제, 성장 한계
결정: 종합 금융 플랫폼으로 피벗
결과: 핀테크 업계 1위
교훈: 과감한 피벗이 때로는 생존 전략
```
