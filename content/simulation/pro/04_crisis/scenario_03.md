# Pro 시나리오 12: 사이버 전쟁 대응

> 전쟁은 총성 없이 시작됩니다.
> 전력망이 꺼지고
> 금융 시스템이 멈추며
> 병원 시스템이 마비됩니다.
> 공격자는 보이지 않고
> 피해는 실시간으로 확산됩니다.
> 반격할 것인가, 방어할 것인가?
> 누가 했는지도 모르는 전쟁에서
> 당신은 어떤 결정을 내릴 것인가?

### 배경: 국가 사이버 공격

```
시점: 화요일 오전 6시 30분
      (출근 시간 직전)

발생 상황:
  오전 6:15 전력망 이상 신호
  오전 6:28 전국 14개 변전소 동시 다운
  오전 6:30 코리아나 전력 15% 차단
  오전 6:35 금융 결제 시스템 오류
  오전 6:40 국방부 내부망 침투 경보
  오전 6:45 주요 방송사 방송 중단
  오전 6:50 청와대 국가사이버안전센터 비상 소집

현재 피해 (7:00 기준):
  정전: 서울·경기·인천 지역
        300만 가구 (부분 정전)
  금융: 카드 결제 오류
        ATM 40% 작동 중단
  교통: 서울 지하철 2·3·4호선 운행 중단
  의료: 병원 3곳 EMR 시스템 마비
  통신: 일부 기지국 불안
  국방: 내부 정보망 침투 시도 감지
```

## 공격 분석 (초기, 불완전)

```

  멀티벡터 동시 공격
  각 시스템 취약점 사전 파악
  정교한 사전 계획 추정

귀속 (Attribution) 현황:
  IP 추적: 동남아 경유지 3개국
  악성코드 패턴: 과거 북한·러시아 연계 그룹과 유사
  하지만 확정 불가 (모방 가능)

과거 유사 사례:
  2021년 한국 원전 해킹
  2022년 우크라이나 전력망 공격
  2023년 대만 정부 사이트 디도스

현재 공격 진행 중:
  2차 공격 징후:
    은행 메인 서버 침투 시도
    항공 관제 시스템 탐색 감지
    댐 제어 시스템 접근 시도

## 참여 팀 구성

### 팀 A: 국가사이버안전센터 (NCSC) (3-4명)

```
구성:
  국가사이버안전센터장 (팀장)
  기술 대응팀장
  위협 분석관
  법률 자문

역할:
  사이버 공격 탐지·분석·대응 총괄
  각 기관 기술 지원
  공격 귀속 분석
  사이버 방어 조율

결정 권한:
  사이버 위기 경보 단계 격상
  인터넷 긴급 차단 (부분)
  취약 시스템 강제 격리
  반격(해킹백) 기술 승인 요청

비공개 정보:
  공격자 귀속:
    내부 분석: 80% 북한 연계 추정
    하지만 확정 증거 없음
  취약 시스템 목록:
    항공 관제: 패치 미적용 취약점 확인
    댐 제어: 6개월 전 침투 흔적 발견
    원전: 격리망 우회 가능성 검토 중
  반격 역량:
    기술적 반격 능력 보유
    하지만 귀속 확정 없이 반격 시
    무고한 제3국 피해 위험
```

## 사이버 안보 핵심 개념 (참고)

```
귀속 (Attribution):
  사이버 공격의 주체를 특정하는 것
  기술적·정치적으로 매우 어려움
  "스모킹 건" 없이 99% 확신도 부족할 수 있음

사이버 억지 (Cyber Deterrence):
  핵 억지와 달리 작동이 불명확
  보복 능력 과시 vs 취약점 노출 딜레마

해킹백 (Hack Back):
  공격에 대한 사이버 반격
  법적으로 대부분 국가에서 불명확
  국제법 적용 여부 논쟁

탈린 매뉴얼:
  NATO 사이버전 국제법 해석 문서
  사이버 공격의 무력 사용 기준
  자위권 적용 조건
```

## 타임 프레셔 시뮬레이션

```
시간별 상황 변화:
  T+0: 공격 감지
  T+30분: 피해 확산 (정전·금융)
  T+1시간: 병원 마비
  T+2시간: 국민 패닉 시작
  T+4시간: 복구 시작 or 추가 공격
  T+8시간: 외교적 압박 시작
  T+12시간: 귀속 분석 중간 결과
  T+24시간: 1차 복구 완료 or 실패
  T+48시간: 전략적 대응 방향 확정

각 시점 결정의 중요성:
  초기 결정이 이후 모든 것을 결정
  정보가 부족해도 결정해야 함
  결정 지연 자체가 결정
```

```

### 사례 2: 에스토니아 사이버 공격 (2007)

```
상황: 러시아와 외교 갈등 후
공격: 정부·은행·언론 DDos
결과: 3주간 온라인 인프라 마비
대응: NATO 지원 + 국제 협력
교훈:
  소국의 사이버 취약성
  집단 방어 (NATO) 필요성
  → 탈린 매뉴얼 제정 계기
```

### 사례 3: 솔라윈즈 공급망 공격 (2020)

```
공격: 러시아 SVR (정보기관)
방식: IT 소프트웨어 업데이트에 악성코드 삽입
피해: 미국 정부·기업 수천 곳
교훈:
  공급망 공격의 광범위성
  탐지까지 9개월 소요
  "보이지 않는" 공격의 위험
```
