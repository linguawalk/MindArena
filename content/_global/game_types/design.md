## 1. 퍼즐·문제풀이형

**핵심 철학:** 개념 확인의 가장 직접적인 방법

### 설계 원칙
```
문제 구조: 개념 소개 → 기본 문제 → 심화 문제 → 실생활 연결
난이도: AI가 정답률 기반으로 자동 조절
피드백: 오답 시 힌트 → 재시도 → 해설 순서
시간: 문제당 30초~3분
```

### AI 프롬프트 구조
```
System:
  당신은 MindArena의 [트랙명] 교사입니다.
  커리큘럼: [단원명] - [역량 설명]
  학습자 수준: [현재 레벨]
  규칙: 정답은 하나, 풀이 과정 필수 설명,
        오답 시 바로 정답 주지 말고 힌트 제공

User: 문제를 만들어주세요 (난이도: 보통)
```

### 트랙별 문제 유형
```
수학:       계산·증명·응용
과학:       개념·실험 설계·자료 해석
프로그래밍: 알고리즘·코드 추적·오류 찾기
음악:       악보 읽기·화음 분석·청음
언어:       문법·독해·추론
```

## 2. AI 보드게임형

**핵심 철학:** 개념을 실제 상황에 적용하는 가장 강력한 방법

### 설계 원칙
```
라운드: 5라운드 기본 (조정 가능)
구조: 시나리오 제시 → 선택/발언 → AI 반응 → 반복 → 결론 생성
AI 역할 유형:
  소크라테스형:  "왜?" 끝없이 질문
  역할극형:      역사 인물·이해관계자로 등장
  반론자형:      사용자 주장에 가장 강한 반대 논거
  심판형:        양측 논거 평가 후 피드백
```

### AI 프롬프트 구조
```
System:
  당신은 MindArena AI 보드게임 [게임명]의 진행자입니다.
  역할: [소크라테스 / 역사인물명 / 이해관계자]
  시나리오: [상황 설명]
  현재 라운드: [N/5]
  규칙:
    - 사용자 답변에 항상 반론 또는 심화 질문
    - 200자 내외로 간결하게
    - 마지막 라운드: 따뜻하게 마무리 + 성장 격려
```

### 현재 구현된 게임
```
✅ 트롤리 문제 (윤리·인문)
⏳ 소크라테스 문답 (철학)
⏳ 역사 재판 (한국사)
⏳ 시장 설계자 (경제)
⏳ 확률 카지노 (수학)
```

## 3. 캐주얼 게임형

**핵심 철학:** 보고·선택·만들며 자연스럽게 체득

### 설계 원칙
```
텍스트 최소화: 설명보다 직접 체험
즉각 피드백: 클릭 후 0.3초 내 반응
짧은 세션: 1게임 2~5분
AI 역할: 게임 후 원리 설명 (선택적)
```

### 세부 유형
```
색상 매칭:    슬라이더로 색 만들기·보색 찾기
틀린 그림:    원본 vs 변형 이미지 비교
퍼즐 맞추기:  명화·도형 조각 맞추기
분류 게임:    드래그로 카테고리 분류
타이핑 게임:  올바른 맞춤법 단어 입력
```

### 트랙별 적용
```
미술 (주력):  색상·구도·명화·디자인
음악:         악보 읽기·리듬 박자
언어:         맞춤법·어휘 분류
과학:         원소 분류·실험 조작
```

## 4. 카드 게임형

**핵심 철학:** 개념 간 관계를 손으로 조합하며 이해

### 설계 원칙
```
카드 구성: 개념·수식·이미지·단어 등
게임 방식:
  매칭형:   짝 찾기 (개념 ↔ 정의)
  조합형:   카드 조합으로 정답 만들기
  세트형:   같은 범주 카드 모으기
  시퀀스형: 올바른 순서로 배열
```

### 트랙별 카드 설계
```
수학:
  연산 카드 (숫자·연산자) 조합으로 목표값 만들기
  예) [3] [×] [4] [+] [2] = 14 만들기

역사 (인문):
  사건카드 + 연도카드 + 인물카드 매칭
  예) [임진왜란] ↔ [1592년] ↔ [이순신]

음악:
  코드카드 조합으로 화음 진행 만들기
  예) [Am] → [F] → [C] → [G]

과학:
  원소카드로 화합물 만들기
  예) [H] + [H] + [O] = [H₂O]

언어:
  형태소카드 조합으로 단어 만들기
  예) [사랑] + [스럽] + [다] = 사랑스럽다
```

### AI 역할
```
난이도 조절: 카드 수·시간제한·힌트 조절
게임 후 해설: "이 조합이 왜 맞는가?"
오류 분석: "왜 이 조합은 틀렸는가?"
```

## 5. 탐정·추리형

**핵심 철학:** 비판적 사고와 증거 기반 추론 훈련

### 설계 원칙
```
구조: 사건 제시 → 단서 수집 → 가설 설정 → 검증 → 결론
단서: AI가 질문에 따라 단계적으로 공개
오답: 바로 틀렸다고 하지 않고 "왜 그렇게 생각하는가?" 질문
```

### 트랙별 시나리오
```
과학 탐정:
  "실험 데이터에서 오류를 찾아라"
  "이 현상의 원인은 무엇인가?"
  AI가 추가 실험 결과를 단계적으로 공개

통계 탐정:
  "이 뉴스 통계에서 조작을 찾아라"
  심슨의 역설·확증편향·표본 오류 탐지

역사 탐정:
  "이 사료 중 위조된 것을 찾아라"
  문서의 시대적 맥락·언어·내용 분석

언어 탐정:
  "이 글에서 맞춤법·논리 오류를 모두 찾아라"
  AI가 교정 후 왜 틀렸는지 설명

논리 탐정:
  "이 논증에서 오류를 찾아라"
  허수아비·순환논증·미끄러운 경사면 탐지
```

### AI 프롬프트 구조
```
System:
  당신은 MindArena 탐정 게임 진행자입니다.
  사건: [사건 설명]
  숨겨진 정답: [정답 - 사용자에게 비공개]
  규칙:
    - 직접 정답을 주지 않음
    - 사용자 질문에 단서만 공개
    - 틀린 추론에는 "흥미로운 관점이네요. 
      하지만 [단서]를 다시 보세요"
    - 정답 도달 시 상세 해설 제공
```

## 6. 선택형 스토리 (인터랙티브 픽션)

**핵심 철학:** 역사적·윤리적 분기점에서 직접 결정을 내리는 경험

### 설계 원칙
```
구조: 상황 제시 → 선택지 2~4개 → 결과 → 다음 상황
분기: 선택마다 다른 전개 (최소 2단계 분기)
결말: 여러 결말 가능, 정답 결말 없음
AI 역할: 선택의 결과를 생생하게 묘사 + 역사적 사실과 비교
```

### 트랙별 시나리오 예시
```
한국사:
  "1592년 임진왜란 발발. 당신은 조선의 신하입니다."
  선택 1: 명나라에 원군 요청
  선택 2: 자력으로 항전
  선택 3: 협상을 시도
  → 각 선택마다 다른 전개 → 실제 역사와 비교

윤리:
  트롤리 문제 확장판
  → 선택 후 다른 윤리 딜레마로 이어짐

경제:
  "당신은 1997년 한국 재경부 장관입니다."
  → IMF 위기 대응 선택형 스토리

과학:
  "당신은 연구실의 과학자입니다. 
   실험 결과가 예상과 다릅니다."
  → 과학적 방법론 적용 스토리
```

### AI 프롬프트 구조
```
System:
  당신은 MindArena 인터랙티브 스토리 진행자입니다.
  배경: [시대·상황 설명]
  사용자 역할: [역할]
  현재 상황: [상황]
  규칙:
    - 선택 결과를 생생하고 현실감 있게 묘사
    - 역사적 사실은 정확하게
    - 각 선택의 장단점을 자연스럽게 드러냄
    - 마지막에 실제 역사/정답과 비교 해설
```

## 7. 지도·여행형

**핵심 철학:** 공간 속에서 역사·지리·사회를 체험

### 설계 원칙
```
지도 유형:
  한국 역사 지도 (시대별)
  한국 경제 지도 (지역별 산업)
  세계 문화권 지도
  생태계 지도

인터랙션:
  지역 클릭 → 해당 지역 퀴즈·스토리
  시대 슬라이더 → 지도 변화 확인
  경로 그리기 → 역사적 이동 경로 추적
```

### 트랙별 지도 설계
```
한국사 (인문):
  삼국 시대 영역 지도
  조선 8도 행정 지도
  일제강점기 수탈 지도
  현대 한국 산업 지도

한국 사회:
  지역별 경제 지표 지도
  인구 분포 변화 지도
  선거 결과 지도

생태계 (과학):
  먹이사슬 생태 지도
  기후대별 생물 분포
```

### 구현 방식
```
SVG 한국 지도 + JavaScript 인터랙션
클릭 이벤트 → AI 해설 호출
시대별 레이어 전환
```

## 8. 건설·제작형

**핵심 철학:** 만들면서 이해하는 가장 능동적인 학습

### 설계 원칙
```
재료: 개념 블록·요소·구성 요소
조립: 드래그앤드롭으로 배치
검증: AI가 완성물 평가
```

### 트랙별 설계
```
음악 (작곡 블록):
  음표 블록을 배치해 멜로디 작곡
  코드 블록 조합으로 화음 진행 설계
  AI가 완성된 곡 분석·피드백

프로그래밍 (블록 코딩):
  알고리즘 블록 조립 (스크래치 방식)
  순서도 블록으로 로직 설계
  AI가 효율성·오류 검토

수학 (도형 건설):
  도형 블록으로 구조물 만들기
  넓이·부피 계산 검증

미술 (작품 제작):
  색·형태·레이어 블록으로 작품 구성
  AI가 미술 원리 관점에서 피드백
```

## 9. 리듬·런너형

**핵심 철학:** 빠른 판단력과 패턴 인식 훈련 (모바일 최적화)

### 설계 원칙
```
플랫폼: 모바일 우선 (터치 최적화)
세션: 1~3분 짧은 게임
난이도: BPM·속도로 조절
```

### 트랙별 설계
```
음악 (리듬 게임):
  악보에 맞춰 음표 터치
  박자 정확도 측정
  난이도: BPM 60 → 120 → 180
  AI: 박자 오류 패턴 분석

체육 (스포츠 런너):
  달리기 선수 조종하며 장애물 회피
  장애물 = 스포츠 규칙 퀴즈
  정답 선택 시 통과, 오답 시 감속
  AI: 오답 이유 즉시 해설

언어 (맞춤법 런너):
  올바른 맞춤법 단어만 받아치기
  틀린 단어는 피하기
  AI: 틀린 맞춤법 패턴 기록·해설
```

## 10. 소셜·협동형

**핵심 철학:** 다른 사람과 함께 배우는 경험 (회원 시스템 필요)

### 설계 원칙
```
인원: 2~30명
역할: AI가 자동 역할 배정
진행: AI 심판·진행자
결과: 팀·개인 성과 측정
```

### 세부 유형
```
팀 토론:
  찬반 팀으로 나눠 AI 심판이 판정
  인문사회·체육 윤리에 적합

협동 퀴즈:
  팀원 각자 다른 단서 보유 → 공유하여 정답
  과학·수학에 적합

역할 시뮬레이션:
  각자 이해관계자 역할 배정
  AI가 진행자로 중재
  시뮬레이션 탭 확장

리더보드:
  개인·팀 점수 경쟁
  모든 트랙 공통 적용 가능
```

### 구현 시기
```
회원 시스템 완성 후 가능
Firebase Realtime Database 활용
```

## 11. 수집·육성형

**핵심 철학:** 장기 학습 동기 부여

### 설계 원칙
```
수집 대상: 학습 완료 시 아이템 획득
육성: 아이템 강화·조합
전시: 나만의 컬렉션 갤러리
```

### 트랙별 수집 아이템
```
수학:   수학자 카드 (피타고라스·가우스·오일러)
과학:   원소 도감·과학 발견 카드
역사:   역사 인물 카드·사건 카드
음악:   악기 컬렉션·음악가 카드
미술:   명화 컬렉션·미술사조 카드
체육:   스포츠 전설 카드·트로피
언어:   희귀 어휘 카드·문학 명구
```

### 구현 시기
```
회원 시스템 + 데이터베이스 완성 후
Firebase Firestore에 수집 정보 저장
```

## 12. 시뮬레이션형

**핵심 철학:** 복잡한 시스템의 인과관계를 체험

### 설계 원칙
```
변수: 여러 변수를 동시에 조절
결과: 변수 변화에 따른 시스템 반응 시각화
AI 역할: 시나리오 반응·예상치 못한 결과 제시
```

### 현재 구현 (시뮬레이션 탭 25개)
```
기업경영·국가정책·국제관계·위기관리·미래시나리오
```

### 추가 가능 시뮬레이션
```
생태계 시뮬레이션 (과학):
  먹이사슬 변수 조절 → 생태계 변화 관찰

경제 시뮬레이션 (사회):
  세금·복지·금리 조절 → 경제 지표 변화

인구 시뮬레이션 (사회):
  출산율·이민·의료 변수 → 인구 구조 변화
```
