# MindArena 문제 작성 형식 가이드

> 모든 콘텐츠 파일의 표준 형식입니다.
> 이 형식을 지켜야 나중에 사이트로 자동 변환이 가능합니다.

---

## 파일 헤더 (Front Matter)

모든 콘텐츠 파일은 다음 헤더로 시작합니다.

```
---
type: [number·logic·language·probability·spatial·pattern·strategy·mechanism·creative]
level: [1-15]
title: [단계 제목]
difficulty: [입문·기초·발전·심화·마스터]
thinking: [핵심 사고 역량 (쉼표로 구분 가능)]
problems: [문제 수]
---
```

### 헤더 작성 예시

```
---
type: number
level: 7
title: 숫자 패턴 찾기 — 다단계 추론
difficulty: 발전
thinking: 다단계 추론 · 수학적 구조 파악
problems: 10
---
```

## 파일 제목과 소개

헤더 다음에 파일 제목과 단계 소개를 씁니다.

```markdown
# [유형명] [단계번호]단계 — [단계 부제]

> [단계 소개 문장 1-3줄]
> 학습자에게 이 단계에서 무엇을 배우는지 간결하게 안내합니다.
```

### 예시

```markdown
# 숫자 퍼즐 7단계 — 다단계 추론

> 발전 구간에 오신 것을 환영합니다.
> 한 번에 답이 보이지 않습니다.
> 여러 단계를 거쳐 추론해야 합니다.
```

## 문제 형식 (표준)

각 문제는 다음 구조를 따릅니다.

```markdown
---

## 문제 N

[문제 내용]
[필요시 표·코드블록·그림 설명 포함]

### 힌트 1
[첫 번째 힌트 — 방향 제시]

### 힌트 2
[두 번째 힌트 — 더 구체적 단서]

### 정답
[정답]

### 해설
[풀이 과정 + 교육적 설명]
[관련 개념·용어 설명 포함 권장]
```

### 문제 작성 규칙

| 항목 | 규칙 |
|------|------|
| 문제 번호 | ## 문제 1 ~ ## 문제 10 |
| 구분선 | 각 문제 앞에 `---` 필수 |
| 힌트 수 | 원칙적으로 2개 (마스터 구간은 최소화) |
| 힌트 방향 | 힌트1=방향제시, 힌트2=구체적단서 |
| 정답 | 간결하게 (한 줄~세 줄) |
| 해설 | 풀이과정 + 개념 설명 포함 |

## 단계 마무리 형식

마지막 문제 다음에 다음 단계 안내를 씁니다.

```markdown
---

## [단계번호]단계 완료 후 안내

> 🎉 [단계번호]단계를 완료했습니다!
> [이 단계에서 배운 핵심 내용 요약]
> [다음 단계 예고 또는 동기 부여 문장]
```

## 콘텐츠 유형별 추가 규칙

### 숫자 퍼즐 (number)
- 수식은 일반 텍스트로 표현 (예: `n² + 1`)
- 수열은 `→` 또는 `,`로 구분
- 표가 필요한 경우 마크다운 표 사용

### 논리 퍼즐 (logic)
- 조건은 번호로 나열 (`조건 1:`, `조건 2:`)
- 진리표는 마크다운 표 사용
- 경우의 수 나열 시 기호 통일 (①②③ 또는 A B C)

### 언어 퍼즐 (language)
- 한국어 고유 표현은 큰따옴표로 강조 (`"눈치"`)
- 한자 표기 시 괄호 병기 (눈치(勘))
- 발음 표기 시 대괄호 사용 ([국물] → [궁물])

### 확률 퍼즐 (probability)
- 경우의 수는 수형도 또는 표로 표현
- 분수는 `P(A) = n/m` 형식
- 조합 기호: `C(n,r)` 형식

### 공간 퍼즐 (spatial)
- 도형은 텍스트 아스키아트 또는 설명으로
- 방향은 상·하·좌·우·앞·뒤로 통일
- 좌표는 (행, 열) 형식

### 보드게임
- 게임 규칙은 `규칙:` 항목으로 먼저 정리
- 예시 국면은 텍스트 보드로 표현
- 전략 분석은 `전략:` 항목으로 분리

### 시뮬레이션
- 시나리오 배경은 `## 배경` 섹션으로
- 역할·입장은 `## 참여자` 섹션으로
- 토론 주제는 `## 핵심 쟁점` 섹션으로

## 마크다운 사용 규칙

### 제목 계층
```
# 파일 제목 (1개만)
## 문제 번호 / 주요 섹션
### 힌트·정답·해설 / 하위 항목
#### 세부 항목 (필요시)
```

### 강조
- **굵게**: 핵심 용어·정답·중요 개념
- *기울임*: 외래어·인용
- `코드`: 수식·기호·코드

### 목록
- 순서 없음: `-` 사용
- 순서 있음: `1.` `2.` 사용
- 중첩: 두 칸 들여쓰기

### 표
```markdown
| 열1 | 열2 | 열3 |
|-----|-----|-----|
| 내용 | 내용 | 내용 |
```

### 코드 블록
수식·예시 코드는 백틱 3개로 감쌉니다.

## 좋은 문제·나쁜 문제 예시

### 좋은 문제 ✅
```markdown
## 문제 3

**다음 수열의 규칙을 찾아 10번째 항을 구하세요.**

3, 7, 11, 15, 19 ...

### 힌트 1
앞뒤 수의 차이를 구해보세요.

### 힌트 2
n번째 수 = 3 + (n-1) × 4

### 정답
39

### 해설
4씩 커지는 등차수열입니다.
n번째 항 = 3 + (n-1) × 4
10번째 항 = 3 + 9 × 4 = 39.
```

### 나쁜 문제 ❌
```markdown
## 문제 3

다음 답은?
1 2 3 4 5 ?

### 정답
6

### 해설
그냥 1씩 더하면 됩니다.
```

나쁜 이유:
- 문제가 너무 단순하고 설명이 없음
- 힌트가 없음
- 해설이 교육적이지 않음
- 제목·맥락이 없음
