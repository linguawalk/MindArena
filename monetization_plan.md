## 5. 결제 시스템 설계

### Toss Payments (국내)

```
대상: 한국 사용자
결제 수단: 카드·계좌이체·토스페이·카카오페이·네이버페이
구현:
  @tosspayments/payment-widget SDK
  Vercel Serverless Function으로 결제 처리
  Firebase에 구독 정보 저장

흐름:
  구독 버튼 클릭
    ↓
  Toss Payments 결제창
    ↓
  결제 완료 → Webhook → Firebase 구독 업데이트
    ↓
  해당 트랙 잠금 해제
```

### Stripe (글로벌)

```
대상: 해외 사용자
결제 수단: Visa·Mastercard·PayPal 등
구현:
  Stripe Checkout (최소 코드)
  Stripe Webhooks → Firebase 구독 업데이트

가격: USD 기준 (환율 자동 적용)
  개별 트랙: $7.99/월
  전체 패키지: $44.99/월
```

## 6. 기술 구현 계획

### 인증 시스템 (Firebase Auth)

```
가입 방법: 이메일 + 구글 로그인
저장 정보:
  uid, email, displayName
  subscriptions: { math: true, science: false, ... }
  freeTrialUsed: { math: true, ... }
  progress: { math: { unit: 3, lesson: 2 }, ... }
```

### 접근 제어 로직

```javascript
// 콘텐츠 접근 권한 확인
function canAccess(trackId, unitNumber) {
  if (!isLoggedIn()) {
    return unitNumber === 0; // 비회원: 미리보기만
  }
  if (unitNumber === 1) {
    return true; // 로그인 사용자: 단원 1 무료
  }
  return hasSubscription(trackId); // 단원 2~: 구독 필요
}
```

### Vercel Serverless Functions 추가 필요

```
api/
  chat.js          ← 기존 (Claude API)
  payment-toss.js  ← Toss Payments 처리
  payment-stripe.js← Stripe 처리
  webhook-toss.js  ← Toss Webhook 수신
  webhook-stripe.js← Stripe Webhook 수신
  subscription.js  ← 구독 상태 확인
```

## 8. 구현 우선순위

```
Phase 1 (즉시): 접근 제어 UI
  → 잠금 화면 표시
  → "구독 필요" 안내
  → 아직 실제 결제 없음 (베타 무료 운영)

Phase 2 (1개월): 인증 시스템
  → Firebase Auth 연동
  → 로그인/회원가입
  → 구독 상태 저장

Phase 3 (2개월): 결제 시스템
  → Toss Payments 먼저 연동
  → 실제 구독 판매 시작

Phase 4 (3개월): Stripe 추가
  → 글로벌 결제 지원
  → 다국어 UI 추가
```
